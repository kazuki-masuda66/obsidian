# Eagle X-Sales feedback-各リージョンからのフィードバック

## 概要
Eagle XプロジェクトのSales feedbackに関するメモ。EAS、EAS-JP、SAS、EUA、LAからのフィードバック、Direction to Sales（User-friendly UI/UX、Sales manual work削減、GHQサポート強化、付加価値データ提供）について記録しています。

## 内容

### EAS
- Rate structureをPricer Remarkに戻して欲しい
- さらなる短縮化、Pricerともっと交渉するための時間が必要

### EAS -JP
- Spread sheetのロードが遅い(?)
- Pricerの回答時間を短くして欲しい
- Blank laneをやめて欲しい（おそらくTTのこと？）
- 操作、オペレーションのシンプル化
- デモ回数を増加して欲しい（トレーニング？）
- どのように裏側が動いているかわからず、個別に設定が必要だった（TTのこと？）
  - AsIsより1.5倍時間がかかった
- Status listのカラムが多すぎる、誤った入力などを避けるために編集不可にするべき
- OPUS filingにつなげて欲しい
- ONE JマニュアルとGHQマニュアルの内容が食い違ってて混乱した
- ツールがUser friendlyじゃなさすぎる。Tigerの方を使いたい
- Surcharge structureについて営業とPricerの差異を明らかにして欲しい
- Salesが求めている条件とPricerの結果に差異がたくさんあり、それを調整するのに多くの時間がかかった
- プライサーの考えるサーチャージ体系が、営業意向シートと異なることがあった
- サーチャージが間違っていて、やり直さないといけなかった
- 入力する手作業が多すぎる
- チェックすることが多すぎて、別々のフォルダに入力しなければならない
- NVOCCのNAC入札は、このツールは適していない
- Sales Review sheet： 営業がプライサー提出シートやコンソリデーションファイルのすべての情報が正しいかどうかをチェックするのは容易ではない。POR、DEL、DEM&DET条件など、重要な情報を営業が一目で確認できるようにフォーマットを改善した方が良い
- ステータスリストの進捗確認： 情報が多すぎて、自分のPICアカウントとアクセスしたいリンクが見つからない
- レーンがたくさんあるアカウントでは、EagleXはコンバージョンがうまくいっているかチェックするのに時間がかかるため、使用しない方がよい
- 締め切りに十分な期間（3週間）を設定したにもかかわらず、回答が届いたのは締め切りの2日後だった
- Preconversionの結果がうまく反映されていなかった
- 営業とPricerでDirectにコミュニケーションさせて欲しい
- レーン数が少ない顧客にとって、TIGERはより速く、より正確である（レーン数が増えれば増えるほど、Eagle-Xの優位性は高まる）
- Simplified formatのほうがCustomer formatよりよい（Customer formatは時間がかかりすぎる）
- メール件名の自動変更
- GHQ/RHQのサポートは非常に限られていた
- どのサーチャージ条件をプライサーが利用したか明確にする必要がある
- Status listが複雑
- スプレッドシートが多すぎたり、手順が複雑だったりすると、作業時間が短縮できない
- ファイルが多すぎる、画面が多すぎる。自分がどこにいるのかわからない。（一つの画面で作業したい。工程数を減らしたい）
- どのEメール・タイトルも同じように見えるので混乱する
- SalesはCSTと同じConversionをできる
- Eagle Xでは交渉の履歴が残らない
- Eagle-Xの進捗状況についてメールが届くが、内容が少し難しい。わかりやすくしてほしい
- Pricing不要レーン（オファーなしレーン）のリクエスト方法がわかりにくい

### Direction to Sales

#### 1. User-friendly UI/UX
- **Process Simplification**:
  - Sales will no longer have to be aware of onshore and offshore processes (Eagle X will integrate the process)
  - Sales can upload customer sheets to Initial Check In directly (No need to check Status list and )
  - The creation of surcharge information by CST will be stopped for next season (Sales will simply communicate the rate structure with Pricers)
- **UI improvement**:
  - Eagle X will simplify the Email header / content
  - Eagle X will simplify Status list -> show only related columns and create Region-wise list which will provide more region dedicated information (Easy to check the status)
  - Eagle X will work on reducing the number of inputs and sheets

#### 2. Reduce sales manual work
- **TT information**:
  - Sales will no longer fill in TT information (Eagle X will complete all the blank column)
    - show TT logic fully
- **Award**:
  - Sales doesn't need to create CVA format manually (Eagle X convert Nomination sheet to CVA format)
  - Eagle X tenders from Japan have gone through this process in over 100 cases and have achieved more efficiency in the Award process
- **Filing**:
  - Sales doesn't create filing data from scratch (Eagle X will create the draft data from Pricing and Nomination data)

#### 3. More support from GHQ
- More Eagle X training
- More direct communication/support between Sales and Eagle X team
- More useful contents through Eagle X portal

#### 4. Provide value-added data
- Eagle X will provide Tender data through ONE Force/Looker
- Sales can utilize potential data freely through ONE Force/Looker
