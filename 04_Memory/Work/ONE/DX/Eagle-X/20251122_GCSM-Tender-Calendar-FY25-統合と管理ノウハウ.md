# GCSM Tender Calendar FY25 - 統合と管理ノウハウ

## 概要

GCSM（Global Commercial & Service Management）が管理するFY25 Tender Calendarと、Eagle X Status Listの統合について。全地域のTender情報を一元管理するためのプロセス。

## 背景

- **日付**: 2024年6月21日 - 7月15日
- **発信者**: Daisuke Tsunematsu (GCSM)
- **受信者**: Koji Nakai, Kazuki Masuda, Flourance Tan
- **目的**: FY25 Tender Calendarの共有と、Eagle X Status Listとの統合

## 主要ポイント

### 1. Tender数の内訳（2024年6月21日時点）

- **合計**: 1,078件のTender（GKA/RKA/LKA/ONECare offshore & onshore tenders含む）
- **地域別内訳**:
  - NAM: 355件（うち225件確定、130件CRO保留）
  - EAS: 319件（確定）
  - EUA: 249件（確定）
  - LAT: 93件（確定）
  - SAS: 57件（確定）
  - WAS: 5件（確定）

### 2. 最終確定数（2024年7月12日時点）

- **合計**: 1,073件（5件削減）
- **変更内容**:
  - NAM: 351件（4件削除）
  - EAS: 318件（1件重複削除）
  - EUA: 249件（変更なし）
  - LAT: 93件（変更なし）
  - SAS: 57件（変更なし）
  - WAS: 5件（変更なし）

### 3. BCO Tenderの定義

GCSMが定義する「BCO Tender」の基準（Tender Calendarへの報告が必要）：

1. **Contract Type**: BCO / Direct / Shipper's Association（NVO Name Accountを除く）
2. **Contract Period**: 1年以上（1H/2H 6ヶ月レビューを含む）
3. **Formal Invitation**: 顧客がポータル、Excel、またはTender documentsで正式な招待を提供
4. **Tender Schedule**: 顧客がTender schedule / timelineを提供

### 4. Eagle Xとの統合

- **統合の目的**: Tender Calendar（GCSM）とStatus List（Eagle X）を統合し、全Long termアカウントを1つのファイルで管理
- **統合完了目標**: 2024年6月末または7月初旬
- **統合プロセス**: 
  1. GCSMが各RHQからTender候補を収集
  2. Eagle X/GCSMが統合Status Listを完成
  3. Pre-conversion studyの実施

## 実務上の注意点

### 1. Pre-conversion Study

- **対象**: Early birdアカウント約280件
- **課題**: Pre-conversionを開始するまでに時間がかかる見込み
- **対応**: Customer Masterの更新を優先し、OPUS Contractは後回しでも可

### 2. Status Listの準備

- **FY25 Go Live**: FY25 Go Live用のStatus Listを準備
- **Test Pre-Study**: Test Pre-Study用のStatus Listを準備
- **優先順位**: 月別にアカウントリストを優先順位付けし、進捗を監視

### 3. TSTとの連携

- **Pre-study開始**: TSTとのPre-studyを可能な限り早期に開始
- **GCSM Tender Teamの参加**: TST TenderのPre-study計画と進捗監視には、GCSM Tender Teamを含める必要がある
