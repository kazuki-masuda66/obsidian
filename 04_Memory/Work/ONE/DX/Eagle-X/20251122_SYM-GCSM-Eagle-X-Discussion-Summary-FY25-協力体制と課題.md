# SYM / GCSM Eagle X Discussion Summary FY25 - 協力体制と課題

## 概要

SYM（Strategic Yield Management）とGCSM（Global Commercial & Service Management）のFY25 Tenderに関する議論のサマリー。Eagle Xツール/プロセスの使用を前提とした協力体制の確立と、課題の整理。

## 背景

- **日付**: 2024年5月20-22日
- **発信者**: Koji Nakai (GHQ Eagle X)
- **受信者**: Cary Low, Daisuke Tsunematsu, Carmen Yau (GCSM)
- **目的**: FY25 Tenderの野心的な目標達成に向けた、SYMとGCSMの協力体制の確立

## 主要ポイント

### 1. 議論のポイント

#### 1) Eagle Xアカウントの拡大
- **合意**: SYM / GCSMは、Eagle Xプロセス/ツールを通じてEagle Xアカウントのカバレッジを拡大することに合意
- **目標**: すべてのLong / Medium contract account（GCSMの想定では約1,000アカウント）
- **GCSMのサポート**: GCSMがプロジェクトをサポートし、Sales / offshore team（TST）をEagle Xで進めるようガイド
- **前提条件**: Eagle Xツール/プロセスの強化とPre studyが必須

#### 2) 新しいTenderリストの準備
- **合意**: SYM / GCSMは、必要な情報を地域/アカウント全体でキャプチャする新しいTenderリストを準備することに合意
- **現状**: Tender Calendar（GCSM）/ Status List（Eagle X）
- **目標**: 2024年6月末までに完了

#### 3) GCSM Tender Managementチームのタスクの明確化
- **SYMの要望**: 現在のGCSM Tender Managementチームのタスクを明確化し、以下を確認：
  1. GCSMとEagle Xチームの間で重複タスクがあるか
  2. GCSMがTender Execution（Pre-TenderからPost-Tenderまで）にどのように関与しているか
- **GCSMの対応**: GCSMがプロセス別に現在のタスクを共有
- **SYMの追加要望**: SYMは、いくつかの例を取り上げて、タスク別にGCSMが何をしているかを詳しく説明することを要請（関連ファイルも共有）
- **対応期限**: GCSMに迅速な対応を要請

#### 4) Tender Executionタスクの引き継ぎ
- **SYMの提案**: SYMがTender Execution（少なくともTender stage）の一部のタスクを引き継ぎ、GCSMがSales管理タスク（Planning、顧客プロファイルなど）に集中できるようにする
- **GCSMの懸念**: HKG GCSM Tender ManagementチームがGKAのCKAMの代わりに作業しているため、100%引き継ぐことは困難
- **GCSMの対応提案**: 現在のGCSMタスクと組織を変更せずに、FY25 Tenderのカバレッジ拡大を優先することを提案
- **継続的な議論**: SYM / GCSMは、上記（3）のGCSMからの情報を受領後、さらに議論が必要

#### 5) Offshoreチームのプロセス標準化
- **合意**: SYM / GCSMは、CST（onshore）/ TST（offshore）によるOffshoreチームのプロセス標準化に合意

### 2. GCSMの懸念事項

- **GKAへの対応**: HKG GCSM Tender ManagementチームがGKAのCKAMの代わりに作業している
- **品質維持**: GKAへのオファーに対して高いクオリティが維持できているという自負
- **恒久的な対応**: GCSMとしてCKAMの仕事を恒久的に行うことが果たして良いのかという課題

### 3. FY24 Offshore Accountへの対応

- **合意**: FY24はoffshore accountにおいてもEagle Xを通すことで合意
- **GCSMの意見**: GCSMの意見にも耳を傾けながら、Process / Toolのenhancementを行う必要がある

### 4. 課題と対応

- **Execution部分のシフト**: HKG GCSM Tender Management team（Carmenのチーム）からExecutionの部分をそのままEagle Xにシフトすることに対して、かなり抵抗があった
- **GCSMの役割**: GCSM（Salesのrepresentative）と言いながら、結局はGKAのCKAMの代わりに仕事をしている（TSTのconversionを確認したり、Pricingの内容でトレードにエスカレートしたり等）
- **CVAへのAward volume入力**: CVAへのAward volumeの入力をCKAMの代わりにHKG GCSMが行っている
- **恒久的な対応**: GCSMとしてCKAMの仕事を恒久的に行うことが果たして良いのかという課題

## 実務上の注意点

### 1. 協力体制の確立

- **前提条件**: Eagle Xツール/プロセスの使用を前提とした協力体制
- **GCSMのサポート**: GCSMがSales / offshore team（TST）をEagle Xで進めるようガイド
- **継続的な議論**: GCSMからの詳細情報を受領後、継続的に議論

### 2. Tenderリストの統合

- **統合の目的**: 必要な情報を地域/アカウント全体でキャプチャ
- **完了目標**: 2024年6月末までに完了
- **統合プロセス**: Tender Calendar（GCSM）とStatus List（Eagle X）を統合

### 3. GCSMタスクの明確化

- **重複タスクの確認**: GCSMとEagle Xチームの間で重複タスクがあるか確認
- **Tender Executionへの関与**: GCSMがTender Executionにどのように関与しているか明確化
- **例の共有**: いくつかの例を取り上げて、タスク別にGCSMが何をしているかを詳しく説明

### 4. Offshoreチームのプロセス標準化

- **標準化の対象**: CST（onshore）/ TST（offshore）によるOffshoreチームのプロセス
- **標準化の目的**: プロセスの統一と効率化
