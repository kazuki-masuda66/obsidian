## 実務上の注意点

### 1. Status List更新のタイミング

- **Tender Calendar更新後**: Tender Calendarが更新されたら、速やかにStatus Listを更新
- **削除Tenderの処理**: 削除されたTenderをStatus Listからも削除
- **追加Tenderの処理**: 追加されたTenderをStatus Listに追加

### 2. Tender削除の基準

- **GCSM/GHQ基準**: GCSM/GHQが定義するTender Calendarの基準を満たさなくなった場合
- **確認プロセス**: Tender Calendar更新時に、各Tenderが基準を満たしているか確認
- **削除の通知**: 削除されたTenderについて、関係者に通知

### 3. Tender追加の処理

- **新規Tender**: 2025 Tender Cycleに追加される新規TenderをStatus Listに追加
- **Pre-conversion Study**: 新規追加Tenderについて、Pre-conversion Studyを実施
- **Status List更新**: Pre-conversion Study完了後、Status Listを更新

### 4. Onshore Tenderの管理

- **事前選択**: すべてのOnshore Tenderが事前選択されている
- **RFAs/SC更新**: 2024 Awarded Tenderについて、RFAs/SCの更新が進行中
- **Status確認**: Eagle X Integration Columnで、事前選択状況を確認
