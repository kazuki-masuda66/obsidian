# Cargo Allocation Review Request - Eagle X価値提案ノウハウ

## 概要

Cargo Allocation Review Requestに対するEagle Xの価値提案。Negative CMの発生を防ぎ、精度の高いPricingを実現するためのEagle Xプロセスの重要性。

## 背景

- **日付**: 2024年6月10-11日
- **発信者**: Kenichi Michida (GHQ SYM), Koji Nakai (Eagle X)
- **受信者**: Kensuke Tanaka, Kohei Sawai, Adeline Quek, その他SYM/YMメンバー
- **目的**: D2 TPEBにおけるNegative CMの発生原因の分析と、Eagle Xプロセスによる改善

## 主要ポイント

### 1. Negative CMの発生状況

- **期間**: Week 17-20
- **発生数**: 520 unitsのNegative CMがD2で発生
- **主な原因**: LT（Long Term）からのNegative CMが大部分
- **根本原因**: FY24 LT guideline設定の問題

### 2. 対応策

#### TM（Trade Marketing）の対応
- **改善作業**: FY24 LT guideline設定の改善に取り組む
- **目標**: LTビジネスでNegative CMが発生しないようにする
- **PSS実装**: 6月にPSS（Pricing Support System）を実装し、D2 Negative CMの大部分を解消
- **低収益D2の管理**: 低収益D2をMQC内に維持
- **継続的な監視**: TMがD2の使用と収益推移を継続的に監視

#### Eagle Xの価値提案
- **精度の高いPricing**: Eagle Xプロセスを通すことで、従来よりも精度の高いPricingが可能
- **Expected CMの確認**: Eagle XのPricingツールを使用すれば、LT MRGが間違っていても、ツール内でexpected CMが確認可能
- **Pricerのサポート**: この点において、Pricerをサポートできる
- **LT MRGの改善**: LT MRGにおいても、FAK同様の改善が必要

### 3. Eagle Xのセールスポイント

- **従来プロセスとの比較**: Eagle Xを通していないアカウントがNegative CMをかなり出している
- **ツールの価値**: Eagle XのPricingツールを使えば、MRGが間違っていてもexpected CMが確認できる
- **Rolloutでの強調**: この点をRollout時にしっかり強調する必要がある

## 実務上の注意点

### 1. Negative CMの防止

- **Eagle Xプロセスの活用**: Eagle Xプロセスを通すことで、Negative CMの発生を防止
- **Expected CMの確認**: Pricingツール内でexpected CMを確認し、問題を早期に発見
- **MRG設定の改善**: LT MRGの設定プロセスの改善が必要

### 2. Pricing精度の向上

- **ツールの活用**: Eagle X Pricingツールを活用し、精度の高いPricingを実現
- **CM計算の自動化**: Expected CMの自動計算により、手動計算のエラーを防止
- **リアルタイム確認**: Pricingツール内でリアルタイムにCMを確認可能

### 3. プロセス改善

- **MRG設定の改善**: LT MRGの設定プロセスを改善し、Negative CMの発生を防止
- **FAK同様の改善**: FAKと同様に、LT MRGの改善が必要
- **継続的な監視**: D2の使用と収益推移を継続的に監視
