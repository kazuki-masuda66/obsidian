# Eagle-X-process-for-FY25-Onshore-Offshore統合とReview縮小

## 概要
Eagle X process for FY25に関するメモ。OnshoreとOffshoreの統合、Sales intention sheetとT&Cサマリの統合、Simplified format、すべてのアカウントでInitial check inを通す、Preconversionの必要の決定、Reviewプロセスの縮小、プロセスフローFY25ドラフトの作成について記録されています。

## 内容

### OnshoreとOffshoreは統合する
- **Sales intention sheetとT&Cサマリの統合（プロコンを整理して、関係者と合意する）**
  - Salesに記入させるのか、CSTで記入するのか
    - 顧客の要件なのか、営業のWillなのかもはっきりさせる必要あり
  - 統合したフォーマットをPricing toolの別シートで表示したい
- **オフショアの場合もチェックリストを営業にやらせるか**
- **オフショアの場合も営業にレビューさせるか**
- **TTツールを最終的にマニュアルで全件埋めきるまでをやるか決める**
  - 今期何％が自動でうまって、どれくらいが自動で埋まらなかったか
  - やる場合は具体的なSOPをCSTに提供

### Simplified format
- **Simplified formatのケースはここに書かない**
- **別のスライドで示す**
- **Simplified format → Pricing toolダイレクトを目指す**

### すべてのアカウントでInitial check inを通す
- **Eagle XでPricingを通すかにかかわらず全件ここは通す**
  - 顧客名が事前に登録されていないと時間がかかってしまうので、ここを解消する仕組みが必要
  - 全件通すことで全アカウント数の把握およびNominationの管理（そのアカウントがCVAをアップロードしたかどうか）が可能になる
- **どのオフショア（WNS MNL/WNS INDIA/OMMC）かは自動判定してメールを送る（営業には意識させない）**
- **Workable/Unworkableの自動判定**
- **ファイルはIntial check inでUploadさせる**

### Preconversionの必要の決定
- **JPは全件Preconversion（Checklistマスタの作り込み、営業レビュー）までやっている**
- **他RegionはWorkableかどうかの判定**

### Reviewプロセスの縮小
- **CST Review**
- **Eagle X conversion Review**
- **Eagle X Pricing Review**
- **Sales Review**

### プロセスフローFY25ドラフトの作成
- **1日はPHのため次回のTouch pointを30日の17時－18時にしてそこでReview**
- **2日に道田さんReview**
- **7日に福田さんDiscussion**
- **8日に常松さんDiscussion**
