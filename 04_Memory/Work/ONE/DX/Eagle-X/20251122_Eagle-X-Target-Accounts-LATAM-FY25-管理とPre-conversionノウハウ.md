# Eagle X Target Accounts LATAM FY25 - 管理とPre-conversionノウハウ

## 概要

LATAM地域のFY25 Eagle X Target Accountsの管理と、Pre-conversion Studyの実施について。FY24で処理した51アカウントをベースに、FY25のTarget Accountsを確定。

## 背景

- **日付**: 2024年6月5-6日
- **発信者**: Ryan Dacles (OMMC), Irving Najera (LATAM CSM Tender Management)
- **受信者**: Cassandra Peretto, Yuesheng Chia, Koji Nakai
- **目的**: FY25 Tender seasonのTarget Accountsリストの確定と、Pre-conversion Studyの実施

## 主要ポイント

### 1. FY24 Target Accounts

- **確定アカウント数**: 51アカウント
- **確認時期**: 3ヶ月前に確認済み、1ヶ月前に再確認
- **対象期間**: 2024年4月 - 2025年3月
- **増加率**: 前年（17アカウント）から300%の増加

### 2. FY25 Pre-conversion Study

- **開始時期**: 2024年6月10日
- **対象**: 全地域のTender FY25
- **目的**: 顧客シートがコンバージョン可能か確認し、潜在的なボリュームなどの必要な情報を取得
- **実施者**: Eagle X Conversion unit (CST-C)

### 3. Target Accountsの確定

- **FY24アカウントの継続**: FY24で確定した51アカウントをFY25でも継続使用
- **変更なし**: Tenderリストに変更はないことを確認
- **新規追加の可能性**: テンダーのリリース日が前後する可能性があるため、変更があれば随時報告

### 4. Long Term Contract Accountsへの拡大

- **方針**: 2024年4月以降、すべてのLong term contract accounts（6ヶ月以上）をEagle Xで処理
- **Early Bird対応**: Early BirdからすべてのアカウントをEagle Xアカウントとして処理する必要がある
- **100%対応の目標**: 将来的に100%のアカウントをEagle Xで処理することを目標

### 5. Tender Calendarとの統合

- **統合の目的**: 既存のTender Calendar（GCSM準備）とStatus List（Eagle X準備）を統合し、全Long termアカウントを1つのファイルで管理
- **統合完了目標**: 2024年6月末または7月初旬
- **統合プロセス**: 
  1. GCSMが各RHQからTender候補を収集
  2. Eagle X/GCSMが統合Status Listを完成

## 実務上の注意点

### 1. Pre-conversion Studyの実施

- **対象アカウント**: 新規追加アカウントで、まだPre-conversion Studyを実施していないアカウント
- **実施者**: CST-C (Benhur Dalumpines経由)
- **確認事項**: 
  - 顧客シートがコンバージョン可能か
  - 潜在的なボリューム
  - その他必要な情報

### 2. Status Listへの追加

- **FY25 Status List**: TestとLiveの両方のStatus Listに追加
- **Target Tender Account List**: Target Tender Account Listにも追加
- **Pre-conversion Study**: 新規追加アカウントのPre-conversion Studyを実施

### 3. アカウント管理

- **リリース月の管理**: Column Iにリリース月を記録
- **Eagle Xでの処理**: Column NにEagle Xで処理するかどうかを記録
- **変更の報告**: テンダーのリリース日が前後する場合は、随時報告
