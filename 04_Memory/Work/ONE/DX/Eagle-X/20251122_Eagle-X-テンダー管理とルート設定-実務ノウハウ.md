## Eagle-X Missing Target Routes - Prism VS All Routes

### 背景
- **目的**: End-to-end (E2E) route coverage expansion、特にテンダープライシングのサポート
- **問題**: Product Catalogue（PRD）Set Upから欠落しているPort pairsが存在
- **対象**: Europe/Africa Door TermsとCY inland routes

### 作業内容

#### 1. 欠落ルートの特定
- **方法**: Prism VS All Routesを比較
- **結果**: 欠落しているPort pairsをリスト化

#### 2. トレード別の整理
- **対象トレード**: 7つのトレード
- **方法**: Google Sheetでトレードごとにタブ分け
- **シート名**: "Eagle-X Missing Target Routes - Prism VS All Routes 16/07/2025"

#### 3. 入力フォーマット
- **列A-J**: Port pairs情報（既存）
- **列M-AD**: 必要な入力フォーマット/テンプレート
- **Pricer's Comments**: ドロップダウンから選択
  - Need Route Creation
  - No Service
  - その他（理由がある場合はEagle X Teamに連絡）

#### 4. 期限
- **Deadline**: 2025年7月25日（金）

## EU ETS関連条項

### 条項内容
近年、温室効果ガス排出の管理への重点が世界的・地域的に高まっている。EU、米国、IMOなどによる最新要件に準拠するため、コンテナ輸送をEmissions Trading System（ETS）に追加することにより、総輸送コストの増加が予想される。

### ONEの対応
- **コスト透明性**: 規制が発効した場合、顧客へのコスト透明性を確保
- **SurchargeまたはBase Freight改定**: 最新の動向に応じて、Surchargeの実施またはBase Freightの改定を検討
- **顧客への説明**: Sales Representativesが質問に対応

## ノウハウとベストプラクティス

### テンダー処理のポイント
1. **期限の厳守**: Due Dateを厳守し、早めに提出
2. **通貨の確認**: USD通貨で提出することを確認
3. **計算項目の正確性**: EI、EJ、EKなどの計算項目を正確に計算
4. **条件の明記**: Space & Equipment availabilityに依存することを明記

### ルート設定のポイント
1. **欠落ルートの早期発見**: Prism VS All Routesを定期的に比較
2. **トレード別の整理**: トレードごとに整理し、効率的に処理
3. **入力フォーマットの統一**: 統一された入力フォーマットを使用
4. **期限の管理**: 期限を厳守し、必要に応じてEagle X Teamに連絡

### コミュニケーションのポイント
1. **グループアドレスの活用**: 適切なグループアドレスにCCを入れる
2. **明確な指示**: 作業内容と期限を明確に指示
3. **リソースの共有**: Eagle X PortalやWorkplaceなどのリソースを共有
4. **フィードバックの収集**: Pricerからのフィードバックを収集
