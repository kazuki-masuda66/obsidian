# APとAwardの関係-契約とアカウントプランの変換

## 概要
Award（契約）とAP（Account Plan）の関係について説明します。AwardはAccount Plan Noにコンバートされ、APの作成に使用されます。

## 詳細

### Awardとは
- **Award = 契約（Tender/Bidの結果）**
- Tender（入札）やBid（提案）の結果として決定された契約
- 「最終的に韓国だしLA向け3000本をいただいたレートでお願いします」という契約内容
- Nomination（指名）とも呼ばれる
- 顧客との契約内容（運賃、サービス、期間、数量など）を含む
- **一度決まった契約**として管理される

### AP（Account Plan）とは
- **AP = アカウントプラン（船腹配分計画）**
- 毎月または隔週で作成・更新される**動的な計画**
- 営業担当者が顧客と相談しながら作成
- 週100本くらいを想定しながら、お客さんの生産調整とすり合わせる
- 計画どおりだと問題ないが、来月80本だったりしたら確認する
- Actual bookingは60本だったりする（実際にお客さんが払うのは60本の分）

### APとAwardの関係

#### 基本的な関係
1. **AwardはAccount Plan Noにコンバートされる**
   - Award（契約）が決まると、Account Plan Noが生成される
   - Account Plan NoでAPを管理する

2. **APはAwardに基づいて各顧客への配分を決定**
   - Awardの契約内容（数量、サービス、運賃など）を参考にAPを作成
   - Awardで決まった契約を、月次・隔週の計画に落とし込む

3. **Awardの契約内容がAPに反映される**
   - Awardで決まった運賃、サービス、数量がAPのベースになる
   - ただし、APは実際の需要に応じて柔軟に調整される

#### 違いの整理

| 項目 | Award（契約） | AP（アカウントプラン） |
|------|-------------|-------------------|
| **性質** | 一度決まった契約（静的） | 月次・隔週で更新される計画（動的） |
| **タイミング** | Tender/Bidの結果として決定 | 毎月または隔週で作成 |
| **内容** | 「3000本をこのレートで」という契約 | 「週100本くらいを想定」という計画 |
| **柔軟性** | 契約として固定 | 実際の需要に応じて調整可能 |
| **実績との関係** | 契約数量（例：3000本） | Actual booking（例：60本） |

#### 具体例

**Award（契約）の例**:
```
顧客Aとの契約:
- 韓国→LA向け
- 3000本（TEU）
- レート: $X,XXX/TEU
- 期間: 2025年1月～12月
```

**AP（アカウントプラン）の例**:
```
顧客Aの1月のAP:
- 週100本を想定
- サービス: AX1
- 営業担当者が顧客と相談しながら作成
- 実際のBooking: 60本（実際にお客さんが払うのは60本の分）
```

#### 変換プロセス

```
Tender/Bid（入札・提案）
  ↓
Award（契約決定）
  ↓
Account Plan No生成（Awardからコンバート）
  ↓
AP作成（Awardの契約内容を参考に）
  ↓
月次・隔週で更新（実際の需要に応じて調整）
  ↓
Actual Lifting（実際の船積み実績）
```

## 基本的な使い方

### AwardからAccount Plan Noへの変換

#### 変換の目的
- Awardの契約内容をAPに反映
- Account Plan NoでAPを管理
- 契約とAPの連携

#### 変換手順
1. Awardの契約内容を確認
2. Account Plan Noを生成
3. APにAccount Plan Noを設定
4. Awardの契約内容をAPに反映

### AP作成でのAward活用

#### AP作成時のAward使用
1. Awardの契約内容を確認
2. 契約に基づいて各顧客への配分を決定
3. 契約に基づいてサービス別の配分を決定
4. 契約に基づいて運賃を設定

## 高度な使い方

### 複数Awardの管理
- 複数のAwardを同時に管理
- Award間の整合性を確認
- APへの反映を一括管理

### AwardとActual Liftingの比較
- Awardの契約内容とActual Liftingの比較
- 差異の分析
- 次月のAP作成への反映

### Awardの動的調整
- 実績に基づいてAwardを調整
- 需要変動に応じてAwardを調整
- 顧客の優先度に応じてAwardを調整

## メリット・デメリット

### ✅ Awardのメリット
- 契約内容が明確
- AP作成の参考データとして使用できる
- 契約とAPの連携が可能

### ❌ Awardのデメリット
- 契約内容の変更に対応しにくい
- 複数のAwardの管理が複雑
- 契約と実績の差異が発生しやすい

## 使用例

### 例1: AwardからAccount Plan Noへの変換

**シナリオ**: 顧客BとのTenderで、韓国→LA向け3000本の契約が決まった

```
1. Awardの契約内容を確認
   - 顧客: 顧客B
   - ルート: 韓国→LA
   - 数量: 3000本（年間）
   - サービス: AX1
   - レート: $2,500/TEU

2. Account Plan Noを生成
   - AP-2025-01-B-001（Awardから自動生成）

3. APにAccount Plan Noを設定
   - Account Plan No: AP-2025-01-B-001

4. Awardの契約内容をAPに反映
   - 年間3000本 → 月次250本、週次約62本を想定
   - サービス: AX1
   - レート: $2,500/TEU（契約レート）
```

### 例2: AP作成でのAward活用と実際の調整

**シナリオ**: 1月のAP作成時に、Awardの契約内容を参考にしながら実際の需要に合わせて調整

```
1. Awardの契約内容を確認
   - 年間3000本（月次250本、週次約62本を想定）

2. 営業担当者が顧客と相談
   - 顧客の生産調整を確認
   - 1月は週100本くらいを想定（Awardより多い）

3. APを作成
   - 1月のAP: 週100本を想定
   - サービス: AX1
   - レート: $2,500/TEU（Awardの契約レート）

4. 実際のBooking
   - Actual booking: 週60本（計画より少ない）
   - 実際にお客さんが払うのは60本の分
   - 2月のAP作成時にこの実績を反映
```

### 例3: AwardとAPの差異管理

**シナリオ**: Awardで3000本の契約があったが、実際の需要が異なる場合

```
Award（契約）:
- 年間3000本（月次250本）

1月のAP:
- 週100本を想定（月次約400本）
- 顧客の生産調整により需要が増加

Actual Lifting:
- 週60本（月次約240本）
- 計画より少ない

2月のAP作成時:
- 1月の実績（週60本）を反映
- 顧客と相談して2月の計画を調整
- Awardの契約数量（年間3000本）は維持
```

## よくある問題と解決策

### 問題1: AwardとAPの不整合
- **原因**: Awardの契約内容の変更、APの更新漏れ
- **解決策**: AwardとAPの定期的な整合性確認

### 問題2: 複数Awardの管理
- **原因**: 複数のAwardが存在、管理が複雑
- **解決策**: Awardの一元管理、Account Plan Noでの管理

### 問題3: 契約と実績の大きな差異
- **原因**: 需要予測の誤り、顧客の急な需要変化
- **解決策**: 定期的な実績確認とAwardの見直し

## ベストプラクティス

1. **定期的な確認**: AwardとAPを定期的に確認
2. **整合性確認**: AwardとAPの整合性を確認
3. **Account Plan No管理**: Account Plan NoでAPを管理
4. **契約反映**: Awardの契約内容をAPに正確に反映
5. **ドキュメント化**: 変換プロセスをドキュメント化

## よくある質問

### Q: AwardとAPは同じものですか？
**A**: いいえ、違います。
- **Award**: Tender/Bidの結果として決まった**契約**（静的）
- **AP**: Awardを参考にしながら、月次・隔週で作成される**計画**（動的）

### Q: Awardで決まった数量とAPの数量が違う場合は？
**A**: 問題ありません。APは実際の需要に応じて柔軟に調整されます。
- Award: 年間3000本の契約
- AP: 月次で実際の需要に応じて調整（週100本だったり、週60本だったり）

### Q: AwardからAccount Plan Noへの変換は自動ですか？
**A**: DX taskforceによる自動化が進行中です。現在は一部手動で対応している場合もあります。

### Q: APはAwardなしで作成できますか？
**A**: はい、Non-tender（スポット契約）の場合はAwardなしでAPを作成します。
