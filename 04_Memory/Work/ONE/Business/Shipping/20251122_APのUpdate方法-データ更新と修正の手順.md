# APのUpdate方法-データ更新と修正の手順

## 概要
AP（Account Plan）のUpdate方法について、実際の業務フローに沿って詳細に説明します。最新のVVD確認、Sales Targetの準備、AP期間の定義、データの更新手順を説明します。

## 詳細

### AP Updateの全体フロー
```
1. 最新のVVDを確認
2. Sales Targetフォルダの準備
3. AP期間の定義
4. データの更新
5. 最終結果のConsolidation
```

## 基本的な使い方

### Step 1: 最新のVVDを確認

#### VVDとは
- **VVD = Vessel Voyage Date（船の航海日）**
- 各サービス（AX1、AX2、AX3、AX4）の航海スケジュール

#### 確認方法
- **場所**: `G:\Shared drives\LW AP\11. Data for Prepare AP`
- **絞り込み条件**: 対象のWk（Week）、Service、Domi（Domestic/International）

#### 確認手順
1. 対象フォルダを開く
2. Wk、Service、Domiで絞り込む
3. 最新のVVDを確認
4. AP期間に含まれるVVDを特定

### Step 2: Sales Targetフォルダの準備

#### フォルダ構造
- **場所**: `G:\Shared drives\Latin West Coast\Account plan Year 2024\New AP process\Sales target\2025`
- **構造**: 各月ごとのフォルダを作成

#### 準備手順
1. 対象年のフォルダを確認
2. 対象月のフォルダを作成（存在しない場合）
3. CimonからSales Targetのデータを受領
4. 対象月フォルダに保存
5. データの整合性を確認

#### データ取得のタイミング
- **通常**: 月末
- **推奨**: 20日頃にもらう（早期取得）

### Step 3: AP期間の定義

#### 期間の決定方法
- **基準**: OPUSカレンダーのISO Weekを使用
- **参照**: Googleスプレッドシート
  - URL: `https://docs.google.com/spreadsheets/d/1HWEaV1JMEAq7nk6byFTROOgDZnExphcvFEH_G0-PTkU/edit?gid=1448216665#gid=1448216665`

#### 確認事項
- Sales TargetとBSAの期間が一致しているか
- ISO Weekが正しいか
- 対象サービスが明確か

#### 比較作業
- **CimonのSales Target**と**BSA**を比較
- 大きな差異がある場合は確認が必要

### Step 4: データの更新

#### 更新対象データ
1. Sales Target
2. BSA
3. 各顧客への配分
4. サービス別の配分

#### 更新手順
1. 前月のAPを開く
2. 新しいSales Targetを反映
3. BSAを更新
4. 各顧客への配分を調整
5. サービス別の配分を更新
6. データの整合性を確認

### Step 5: 最終結果のConsolidation

#### Consolidationとは
- 各サービスのExcelシートに転記
- 最終的なAPデータの統合

#### Consolidation先
- **場所**: `G:\Shared drives\Latin West Coast\Account plan Year 2024\Account Plan to sales with no back end data\Account plan 2025`

#### Consolidation手順
1. 各サービスのAPデータを準備
2. Consolidation先フォルダを開く
3. 各サービスのExcelシートに転記
4. データの整合性を確認
5. 最終確認

## 高度な使い方

### 複数サービスの同時更新
- AX1、AX2、AX3、AX4を同時に更新
- サービス間の整合性を確認
- キャパシティの制約を考慮

### 実績データとの比較
- Actual Liftingと計画の比較
- 差異の分析
- 次月の計画への反映

## メリット・デメリット

### ✅ メリット
- 体系的なプロセスで確実に更新できる
- 各ステップで確認ポイントが明確
- エラーを早期に発見できる

### ❌ デメリット
- 手順が多く、時間がかかる
- Manual workが多い
- 複数のフォルダ間での移動が必要

## 使用例

### 例1: 6月APのUpdate
```
5月20日: 最新のVVDを確認
5月21日: Sales Targetフォルダを準備
5月22日: AP期間を定義（ISO Week確認）
5月23日: Sales TargetとBSAを比較
5月24日: データを更新
5月25日: 最終結果をConsolidation
```

## よくある問題と解決策

### 問題1: VVDの確認が遅れる
- **原因**: 最新のVVD情報が更新されていない
- **解決策**: 定期的にVVDを確認、更新を依頼

### 問題2: Sales Targetの取得が遅れる
- **原因**: Cimonからのデータ提供が遅い
- **解決策**: 前月末に依頼、20日頃にリマインダー

### 問題3: データの不整合
- **原因**: 複数のデータソースからの取得
- **解決策**: データの一元管理とバリデーション

## ベストプラクティス

1. **早期準備**: Sales Targetは前月末に依頼
2. **定期的な確認**: VVDを定期的に確認
3. **データ整合性**: 複数のデータソースの整合性を確認
4. **チェックリスト**: 各ステップでチェックリストを使用
5. **ドキュメント化**: プロセスをドキュメント化
