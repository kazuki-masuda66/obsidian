# BSAとAPの関係-船腹保証とアカウントプラン

## 概要
BSA（Berth Space Allocation、船腹保証）とAP（Account Plan）の関係について説明します。BSAはAPの主要コンポーネントであり、各顧客に保証する船腹量を決定します。

## 詳細

### BSAとは
- **BSA = Berth Space Allocation（船腹保証）**
- 各顧客に保証する船腹量
- APの主要コンポーネント
- サービス別に設定される

### APとBSAの関係
- BSAはAPの主要コンポーネント
- APはBSAに基づいて各顧客への配分を決定
- BSAの合計がサービスのキャパシティを超えないようにする

## 基本的な使い方

### BSAの設定方法

#### 設定の考慮事項
1. **Sales Target**: 営業目標
2. **過去の実績**: Actual Lifting
3. **顧客の優先度**: Tier1顧客など
4. **サービスのキャパシティ**: 船の容量

#### 設定手順
1. Sales Targetを確認
2. 過去の実績を確認
3. 顧客の優先度を確認
4. サービスのキャパシティを確認
5. BSAを設定
6. BSAの合計がキャパシティを超えないことを確認

### APでのBSAの活用

#### AP作成時のBSA使用
1. BSAに基づいて各顧客への配分を決定
2. 各サービスへの配分を決定
3. 配分の合計がBSAを超えないことを確認

#### AP更新時のBSA確認
1. 実績とBSAを比較
2. 30%以上の差異がある場合は確認
3. 必要に応じてBSAを調整

## 高度な使い方

### 複数サービスのBSA管理
- AX1、AX2、AX3、AX4のBSAを同時に管理
- サービス間のBSAバランスを考慮
- キャパシティの制約を考慮

### BSAとActual Liftingの比較
- Actual LiftingとBSAの比較
- 差異の分析
- 次月のBSA設定への反映

### BSAの動的調整
- 実績に基づいてBSAを調整
- 需要変動に応じてBSAを調整
- 顧客の優先度に応じてBSAを調整

## メリット・デメリット

### ✅ BSAのメリット
- 顧客に明確な船腹保証を提供
- APの計画が立てやすい
- 営業活動の参考資料として使用できる

### ❌ BSAのデメリット
- 需要変動に対応しにくい
- 固定された配分で柔軟性が低い
- 実績との差異が発生しやすい

## 使用例

### 例1: BSAの設定
```
1. Sales Targetを確認（1000 TEU）
2. 過去の実績を確認（800 TEU）
3. 顧客の優先度を確認（Tier1: 60%, Tier2: 40%）
4. サービスのキャパシティを確認（1200 TEU）
5. BSAを設定（Tier1: 600 TEU, Tier2: 400 TEU）
6. BSAの合計がキャパシティを超えないことを確認（1000 TEU < 1200 TEU）
```

### 例2: APでのBSA活用
```
1. BSAに基づいて各顧客への配分を決定
2. 各サービスへの配分を決定（AX1: 300 TEU, AX2: 200 TEU, AX3: 300 TEU, AX4: 200 TEU）
3. 配分の合計がBSAを超えないことを確認（1000 TEU）
```

## よくある問題と解決策

### 問題1: BSAの合計がキャパシティを超える
- **原因**: BSAの設定ミス、キャパシティの誤認識
- **解決策**: BSAの合計を確認、キャパシティを再確認

### 問題2: 実績とBSAの大きな差異
- **原因**: 需要予測の誤り、顧客の急な需要変化
- **解決策**: 定期的な実績確認とBSAの見直し

### 問題3: 複数サービスのBSAバランス
- **原因**: サービス間のBSA配分が不適切
- **解決策**: サービス間のBSAバランスを考慮

## ベストプラクティス

1. **定期的な見直し**: 実績に基づいてBSAを定期的に見直す
2. **キャパシティ確認**: BSAの合計がキャパシティを超えないことを確認
3. **差異分析**: 実績とBSAの差異を分析
4. **柔軟な調整**: 需要変動に応じてBSAを調整
5. **ドキュメント化**: BSA設定のプロセスをドキュメント化
