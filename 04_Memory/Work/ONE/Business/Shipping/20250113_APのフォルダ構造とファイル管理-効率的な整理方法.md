# APのフォルダ構造とファイル管理-効率的な整理方法

## 概要
AP（Account Plan）のフォルダ構造とファイル管理方法について説明します。効率的な整理方法により、AP関連のファイルを迅速に見つけて管理できます。

## 詳細

### APの主要フォルダ構造

#### 1. Data for Prepare AP
- **場所**: `G:\Shared drives\LW AP\11. Data for Prepare AP`
- **用途**: 最新のVVD確認
- **内容**: VVDデータ、サービスデータ、Domiデータ

#### 2. Sales Target
- **場所**: `G:\Shared drives\Latin West Coast\Account plan Year 2024\New AP process\Sales target\2025`
- **用途**: Sales Targetの保存
- **内容**: 各月のSales Targetデータ

#### 3. Account Plan to sales
- **場所**: `G:\Shared drives\Latin West Coast\Account plan Year 2024\Account Plan to sales with no back end data\Account plan 2025`
- **用途**: 最終結果のConsolidation
- **内容**: 各サービスのExcelシート

#### 4. Convert AP
- **場所**: `G:\Shared drives\LW AP\4_Convert_AP`
- **用途**: Master Constraint Table用データの準備
- **内容**: BKが固めたデータ

#### 5. Space Control Standby
- **場所**: `G:\Shared drives\Latin West Coast\Space Control\AX4\Standby\2025`
- **用途**: Master Constraint Table用データの保存
- **内容**: 各月のMaster Constraint Tableデータ

#### 6. Finalize AP
- **場所**: `G:\Shared drives\LW AP\3_Finalize_AP`
- **用途**: APの最終確認
- **内容**: 最終化されたAPデータ

## 基本的な使い方

### フォルダ構造の理解

#### 階層構造
```
G:\Shared drives\
├─ LW AP\
│  ├─ 3_Finalize_AP\          # APの最終確認
│  ├─ 4_Convert_AP\           # Master Constraint Table用データ
│  └─ 11. Data for Prepare AP\ # VVDデータ
└─ Latin West Coast\
   ├─ Account plan Year 2024\
   │  ├─ New AP process\
   │  │  └─ Sales target\      # Sales Target
   │  └─ Account Plan to sales\ # 最終結果
   └─ Space Control\
      └─ AX4\
         └─ Standby\            # Master Constraint Table
```

### ファイル管理のベストプラクティス

#### 1. 命名規則
- **日付形式**: YYYYMMDD形式を使用
- **サービス識別**: AX1、AX2、AX3、AX4を明示
- **内容識別**: AP、Sales Target、Master Constraint Tableなどを明示

#### 2. フォルダ管理
- **年度別**: 各年度ごとにフォルダを作成
- **月別**: 各月ごとにフォルダを作成
- **サービス別**: サービスごとにフォルダを作成（必要な場合）

#### 3. バックアップ
- **定期的なバックアップ**: 重要なデータは定期的にバックアップ
- **バージョン管理**: 過去のバージョンを保持
- **削除の注意**: 重要なデータは削除前に確認

## 高度な使い方

### ファイル検索の効率化
- **ファイル名での検索**: 日付、サービス、内容で検索
- **フォルダ構造の活用**: 階層構造を活用して検索
- **ショートカットの作成**: よく使うフォルダへのショートカット

### データの整合性確認
- **複数フォルダ間での確認**: 関連するフォルダ間でデータの整合性を確認
- **バージョン管理**: 過去のバージョンと比較
- **エラーチェック**: データのエラーを早期に発見

## メリット・デメリット

### ✅ 効率的なフォルダ構造のメリット
- ファイルを迅速に見つけられる
- データの整合性を確認しやすい
- バックアップとバージョン管理が容易

### ❌ フォルダ構造のデメリット
- 複数のフォルダ間での移動が必要
- フォルダ構造の理解が必要
- ファイルの場所を覚える必要がある

## 使用例

### 例1: 6月APのファイル管理
```
1. Sales Target: G:\...\Sales target\2025\06\
2. AP作成: G:\...\Account plan Year 2024\New AP process\
3. 最終結果: G:\...\Account plan 2025\06\
4. Master Constraint: G:\...\Space Control\AX4\Standby\2025\06\
```

### 例2: ファイル検索
```
検索条件: "2025-06 AP AX4"
検索結果: 
- G:\...\Account plan 2025\06\AX4_AP.xlsx
- G:\...\Space Control\AX4\Standby\2025\06\AX4_Master_Constraint.xlsx
```

## よくある問題と解決策

### 問題1: ファイルが見つからない
- **原因**: フォルダ構造の理解不足、命名規則の不統一
- **解決策**: フォルダ構造の理解、命名規則の統一

### 問題2: データの不整合
- **原因**: 複数のフォルダ間でのデータ管理
- **解決策**: データの一元管理、整合性チェック

### 問題3: バックアップの不足
- **原因**: バックアップの実施不足
- **解決策**: 定期的なバックアップ、バージョン管理

## ベストプラクティス

1. **命名規則の統一**: 一貫した命名規則を使用
2. **フォルダ構造の理解**: フォルダ構造を理解して使用
3. **定期的な整理**: 定期的にフォルダを整理
4. **バックアップ**: 重要なデータは定期的にバックアップ
5. **ドキュメント化**: フォルダ構造をドキュメント化
