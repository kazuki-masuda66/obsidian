# Booking NoとBL NoとContract Noの違い-海運用語の識別番号

## 概要
海運業界で使用される3つの重要な識別番号（Booking No、BL No、Contract No）の違いと関係性について説明します。

## 詳細

### Booking No（予約番号）

#### 定義
- **Booking No = Booking Number（予約番号）**
- 顧客が船社に対して**船積みを予約**した際に発行される識別番号
- 船積みの**予約段階**で発行される
- 1つのBooking Noに対して複数のBL Noが発行される場合がある

#### 特徴
- **タイミング**: 船積み予約時（貨物がまだ積まれていない段階）
- **目的**: 予約の管理、追跡
- **発行**: 船社が顧客の予約を受け付けた時点で発行
- **変更可能性**: 予約内容の変更が可能（キャンセル、数量変更など）

#### 使用例
```
顧客が「来週、横浜からLA向けに10本のコンテナを積みたい」と予約
  ↓
Booking No: ONE20250113001 が発行される
  ↓
このBooking Noで予約を管理
```

### BL No（船荷証券番号）

#### 定義
- **BL No = Bill of Lading Number（船荷証券番号）**
- **Bill of Lading（B/L、船荷証券）**に記載される識別番号
- 船荷証券は**貨物の所有権を表す有価証券**
- 貨物が**実際に船に積まれた後**に発行される
- 1つのBooking Noから複数のBL Noが発行される場合がある（複数のコンテナ、複数の荷主など）

#### 特徴
- **タイミング**: 貨物が船に積まれた後（船積み完了後）
- **目的**: 貨物の所有権の証明、通関書類として使用
- **発行**: 船社が貨物を船に積み込んだ後、BL（船荷証券）を発行
- **法的効力**: 有価証券として法的に重要な書類
- **変更可能性**: 発行後の変更は困難（CODなどの手続きが必要）

#### 使用例
```
Booking No: ONE20250113001 で予約
  ↓
貨物を船に積み込む
  ↓
BL No: ONE20250115001 が発行される（船荷証券に記載）
  ↓
このBL Noで貨物を追跡、通関手続きなどに使用
```

### Contract No（契約番号）

#### 定義
- **Contract No = Contract Number（契約番号）**
- 顧客と船社の間で結ばれた**運賃契約**を識別する番号
- **RFA（Rate Filing Agreement）**や**SC（Service Contract）**などの契約を識別
- 契約期間中、複数のBookingやBLに使用される
- Award（契約）から生成される場合もある

#### 特徴
- **タイミング**: 契約締結時（Tender/Bidの結果として決定）
- **目的**: 運賃契約の管理、料金の適用
- **発行**: 契約が締結された時点で発行
- **有効期間**: 契約期間中（例: 1年間）
- **適用範囲**: 契約で定められた条件（ルート、サービス、運賃など）に適用

#### 使用例
```
顧客AとのTenderで契約が決まる
  ↓
Contract No: NYC0360N21（SC契約番号）が発行される
  ↓
このContract Noで運賃を適用
  ↓
契約期間中、複数のBookingやBLでこのContract Noを使用
```

## 3つの番号の関係性

### 階層構造

```
Contract No（契約番号）
  ↓ 契約に基づいて
Booking No（予約番号）
  ↓ 予約に基づいて貨物を積み込む
BL No（船荷証券番号）
```

### 関係性の詳細

| 項目 | Contract No | Booking No | BL No |
|------|------------|------------|-------|
| **発行タイミング** | 契約締結時 | 予約受付時 | 船積み完了後 |
| **有効期間** | 契約期間中（例: 1年） | 予約から船積みまで | 永続的（貨物の所有権証明） |
| **数量関係** | 1契約 | 1契約から複数の予約 | 1予約から複数のBL |
| **主な用途** | 運賃の適用 | 予約の管理 | 貨物の所有権証明、通関 |
| **変更可能性** | 契約変更が必要 | 予約変更可能 | 変更困難（CODなど） |

### 具体例

#### 例1: 通常のフロー

```
1. 契約締結
   Contract No: NYC0360N21（顧客Aとの1年間の契約）

2. 予約
   Booking No: ONE20250113001（1月13日に10本のコンテナを予約）
   Contract No: NYC0360N21 を適用

3. 船積み
   BL No: ONE20250115001（1月15日に船積み完了、BL発行）
   Booking No: ONE20250113001 から生成
   Contract No: NYC0360N21 で運賃を適用
```

#### 例2: 複数のBLが発行される場合

```
1. 契約締結
   Contract No: NYC0360N21

2. 予約
   Booking No: ONE20250113001（20本のコンテナを予約）

3. 船積み（複数の荷主、複数のコンテナ）
   BL No: ONE20250115001（荷主A、5本）
   BL No: ONE20250115002（荷主B、10本）
   BL No: ONE20250115003（荷主C、5本）
   
   すべて同じ Booking No: ONE20250113001 から生成
   すべて同じ Contract No: NYC0360N21 で運賃を適用
```

#### 例3: Contract Noなしの場合（スポット契約）

```
1. 予約（Contract Noなし、スポット料金）
   Booking No: ONE20250113001

2. 船積み
   BL No: ONE20250115001
   Booking No: ONE20250113001 から生成
   Contract No: なし（スポット料金を適用）
```

## よくある質問

### Q: Booking NoとBL Noは同じですか？
**A**: いいえ、違います。
- **Booking No**: 予約段階で発行（貨物がまだ積まれていない）
- **BL No**: 船積み完了後に発行（貨物が船に積まれた後）

### Q: 1つのBooking Noから複数のBL Noが発行されることはありますか？
**A**: はい、あります。
- 複数のコンテナがある場合
- 複数の荷主がいる場合
- 複数のBLタイプ（Original BL、Surrender BLなど）が必要な場合

### Q: Contract Noは必須ですか？
**A**: いいえ、必須ではありません。
- **Contract Noあり**: 契約運賃を適用（RFA、SCなど）
- **Contract Noなし**: スポット料金を適用（Non-tender）

### Q: Bookingをした場所とBLを発行する場所は同じですか？
**A**: 必ずしも同じではありません。
- Bookingを受けた場所（例: TYOBB）
- BLを発行する場所（例: SELBB）
- お金の支払い地（例: NYCBB）
これらが異なるケースもあります。

### Q: Contract NoとAccount Plan Noの関係は？
**A**: 
- **Contract No**: 運賃契約を識別（RFA、SCなど）
- **Account Plan No**: AP（アカウントプラン）を識別
- Award（契約）からAccount Plan Noが生成される場合がある
- Contract NoとAccount Plan Noは異なる概念だが、関連している
