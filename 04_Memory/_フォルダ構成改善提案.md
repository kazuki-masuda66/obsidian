## 🎯 提案する新しいフォルダ構成（改訂版）

### 問題点の整理
1. **DX/とSystems/の違いが分かりにくい**
   - DX taskforce = 組織レベルのデジタルトランスフォーメーション推進
   - Eagle-X = DX taskforceの一環として進められている具体的なプロジェクト/システム
   - → **Eagle-XはDXの一部なので、統合すべき**

2. **ONE関連が分散していて混乱する**
   - Education/Corporate-Training/ → 研修コンテンツ
   - Technical/Tools/ONE/ → システムツール
   - → **ONE関連を一箇所にまとめるべき**

### 提案する新しい構成（改訂版）

```

│  └─ ONE/                      # ONE社関連の知識を全てここに集約
│     ├─ Business/              # ONE社の業務知識
```

### 構成の考え方

#### Work/ONE/ に集約する理由
- **ONE社関連の知識を一箇所にまとめる**ことで、混乱を避ける
- 業務知識、システム、研修、DXなど、ONE社に関連する全ての知識をここに配置
- 「ONE社のことを知りたい」→ `Work/ONE/` を見れば全て分かる

#### DX/配下に全てのDX関連を集約
- **DX/ = デジタルトランスフォーメーション全般**
  - `Taskforce/` = DX taskforce（組織レベルの推進）
  - `Eagle-X/` = Eagle-Xプロジェクト（DXの一部）
  - `Systems/` = ONE社のシステムツール（OPUS、Tiger、ONE-Quoteなど）
  - `Digital-Marketing/` = デジタルマーケティング関連
- ONE社のDX関連は全て `Work/ONE/DX/` 配下に集約

#### Business/とWork/の使い分け
- **Business/** = 汎用的なビジネス知識（業界・会社を問わない）
  - Sales、Community-Buildingなど
- **Work/** = 特定の会社・組織に関連する知識
  - ONE社関連の知識は全て `Work/ONE/` に集約

## 💡 判断基準（改訂版）

### まず最初に：ONE社関連か？
**質問**: これはONE社に関連する知識か？
- **YES** → `Work/ONE/` 配下の適切なフォルダへ
- **NO** → 汎用的な知識として、Business/、Education/、Technical/ などへ

### Work/ONE/ 配下の判断フロー

```
ONE社関連の知識か？
  ↓ YES
どのカテゴリか？

┌─ 業務知識（AFLA、Shipping）？
│   YES → Work/ONE/Business/Shipping/
│   NO ↓
│
┌─ DX関連？
│   YES ↓
│   ├─ DX taskforce？
│   │   YES → Work/ONE/DX/Taskforce/
│   │   NO ↓
│   ├─ Eagle-Xプロジェクト？
│   │   YES → Work/ONE/DX/Eagle-X/
│   │   NO ↓
│   ├─ システムツール（OPUS、Tiger、ONE Quote）？
│   │   YES → Work/ONE/DX/Systems/[システム名]/
│   │   NO ↓
│   └─ デジタルマーケティング？
│       YES → Work/ONE/DX/Digital-Marketing/
│   NO ↓
│
└─ 研修・教育コンテンツ？
    YES → Work/ONE/Training/[研修名]/
```

### 汎用的な知識の判断

```
ONE社関連でない知識か？
  ↓ YES
どのカテゴリか？

┌─ ビジネス知識（Marketing、Sales、Operations）？
│   YES → Business/[カテゴリ]/
│   NO ↓
│
┌─ 教育・学習知識？
│   YES → Education/[カテゴリ]/
│   NO ↓
│
┌─ 技術知識？
│   YES → Technical/[カテゴリ]/
│   NO ↓
│
└─ 個人関連？
    YES → Personal/[カテゴリ]/
```
